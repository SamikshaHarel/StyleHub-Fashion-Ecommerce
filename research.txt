# 📌 STYLEHUB PROJECT – INTERVIEW NOTES

## 🔷 1. Project Overview

StyleHub is a backend-based fashion recommendation system built using Java Spring Boot.
It allows users to:

* View products (clothes, jewellery)
* Add products to database
* Get outfit recommendations based on place (e.g., Goa, Manali)

👉 Example: If user enters "Goa", system suggests beach outfits.

---

## 🔷 2. Technologies Used

* Java (Core Programming)
* Spring Boot (Backend Framework)
* Maven (Project Management Tool)
* MySQL (Database)
* JPA/Hibernate (ORM for database connection)
* Postman (API Testing Tool)

---

## 🔷 3. What is Maven?

Maven is a build automation and dependency management tool.

It helps to:

* Manage libraries (dependencies)
* Maintain project structure
* Build and run project easily

Main file: `pom.xml`

---

## 🔷 4. Project Structure

com.stylehub
├── controller → Handles API requests
├── service → Contains business logic
├── repository → Communicates with database
├── model → Represents database table

---

## 🔷 5. Model Layer (Product.java)

Represents database table.

Fields:

* id (Primary Key)
* name
* category
* price
* imageUrl
* tags

👉 Tags are important for recommendation logic.

---

## 🔷 6. Repository Layer

Uses JPA to interact with database.

Example:
List<Product> findByTagsContainingIgnoreCase(String tag);

👉 This method searches products based on tags.

---

## 🔷 7. Service Layer

Contains business logic.

Example:

* Add product
* Get all products
* Get products by tag

---

## 🔷 8. Controller Layer

Handles HTTP requests using APIs.

Examples:

* POST → Add product
* GET → Fetch all products
* GET → Fetch products by tag
* GET → Recommendation API

---

## 🔷 9. What is API?

API (Application Programming Interface) is a bridge between frontend and backend.

Example:
http://localhost:8080/api/products

---

## 🔷 10. Why Postman is Used?

Postman is used to test APIs without frontend.

We used it to:

* Send POST request (add product)
* Send GET request (fetch data)

---

## 🔷 11. Database Connection

Configured in application.properties:

spring.datasource.url=jdbc:mysql://localhost:3306/stylehub
spring.datasource.username=root
spring.datasource.password=yourpassword

👉 This connects backend with MySQL database.

---

## 🔷 12. How Project Works Internally

1. User sends request (via browser/Postman)
2. Controller receives request
3. Service processes logic
4. Repository fetches/stores data in database
5. Response is sent back

---

## 🔷 13. Recommendation Feature (Main Feature)

API:
http://localhost:8080/api/products/recommend?place=goa

Logic:

* User enters place
* System matches it with product tags
* Returns matching outfits

Example:
Product tags: beach,goa,summer
User input: goa
→ Product is recommended

JPA म्हणजे काय?

👉 Full form: Java Persistence API

👉 Simple:
JPA = Java मधून database handle करण्याची technique

💡 Without JPA (Old way)

तुला manually SQL लिहावं लागलं असतं:

INSERT INTO product VALUES(...)
SELECT * FROM product;

👉 आणि Java मध्ये खूप code लिहावा लागतो 😓

💡 With JPA (Your way)

तू फक्त class बनवली:

@Entity
public class Product {
    private Long id;
    private String name;
}

👉 बस!
Spring Boot + JPA automatically:

table create करतं
data store करतं
data fetch करतं
🔥 Meaning

👉 Marathi:
JPA म्हणजे Java objects (class) आणि database tables यांच्यामधला bridge
